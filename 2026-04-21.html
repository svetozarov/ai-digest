01 · МОДЕЛИ

Kimi K2.6: 300 агентов в открытом весе

Moonshot AI выпустила Kimi K2.6 — open-weight MoE 1T параметров с 32B активными, вплотную подошедшую к Claude Opus 4.6 и GPT-5.4 на кодинг-бенчах.

K2.6 — апдейт K2.5 (январь 2026). На frontend-design бенчах — 68.6% win+tie rate против Gemini 3.1 Pro. Ключевое отличие: модель обучена координировать до 300 субагентов параллельно, что раньше было прерогативой только облачных систем.

По данным Latent.space, K2.6 обгоняет Composer 2 от Cursor на длинных кодинг-задачах. Модель доступна как open-weight — скачать и запустить локально или на своей инфраструктуре.

Единственная open-weight модель с массовой агентной оркестрацией — потенциальная основа для Personal Wiki и pre-design tool без API-зависимости.
Источник → Latent.space, 21.04

02 · ИНСТРУМЕНТЫ-РАЗРАБОТКИ

Codex Chronicle: экранная память между сессиями

OpenAI добавила в Codex функцию Chronicle — persistent memory, которая следит за экраном и запоминает контекст задач для будущих сессий.

Chronicle читает активные окна разработчика и строит rolling-контекст о проекте. При следующем запросе Codex уже знает, над чем идёт работа. OpenAI признаёт повышенный риск prompt injection при включённом экранном доступе.

Прямой конкурент project context механизму Claude Code — изучение подхода Chronicle даёт понимание направления AI-assisted coding для рабочего процесса с pre-design tool.
Источник → The Decoder, 20.04

03 · МУЛЬТИМОДАЛЬНОСТЬ

Darwin-TTS: эмоции из 3% чужих весов

Darwin-TTS-1.7B-Cross — первая TTS с cross-modal переносом FFN-весов из LLM. При 3% blending модель начинает говорить с эмоцией без единого часа обучения.

Взяли Qwen3-TTS и смешали 3% FFN-весов из большего LLM. При 3% — эмоции появляются, при 5% — усиливаются, при 10% — модель ломается. Весь проект занял один день, ноль GPU-часов на файнтюнинг. Открытая лицензия на Hugging Face.

Прямое применение для VoiceForge: zero-training emotional TTS улучшает naturalness голосового ассистента без дорогостоящего переобучения на RTX 3070 8GB.
Источник → Hugging Face, 15.04

04 · ИНФРАСТРУКТУРА-GPU

Sentence Transformers: текст и чертёж в одном векторе

Sentence Transformers обновился для cross-modal поиска — одна модель кодирует текст и изображения в единое векторное пространство.

Новый релиз добавляет мультимодальные embedding и reranker модели. RAG-системы теперь работают с визуальными документами — PDF с чертежами, смешанным контентом. Cross-modal retrieval без раздельных pipeline-ов.

Для Personal Wiki на Qwen3-8B + Chroma: индексация чертежей и планов вместе с текстом — прямое улучшение pre-design tool для работы с проектной документацией.
Источник → Hugging Face, 09.04

05 · РЕЛИЗ-ПРОДУКТА

GR00T N1.7: робот учится у человека

NVIDIA выпустила Isaac GR00T N1.7 — открытую Vision-Language-Action модель для гуманоидных роботов с коммерческой лицензией.

Обучена на 20,000+ часах human egocentric video. Впервые задокументирован dexterity scaling law: больше человеческих данных — предсказуемо лучше манипуляции. Finger-level control, коммерческое применение разрешено.

Материал для канала @architects_AI: зрелость физического AI достигла точки, где open-source гуманоидная модель готова к производственному применению.
Источник → Hugging Face / NVIDIA, 17.04

06 · МОДЕЛИ

Darwin-27B: обход Opus 4.7 без обучения

Darwin-27B-Opus превосходит Claude Opus 4.7 на GPQA benchmark через cross-model weight transfer — без дополнительного файнтюнинга.

Cross-model weight transfer: перенос весов из более сильной модели в 27B-параметровую без обучения. Результат превосходит Opus 4.7 на GPQA (Graduate-level Q&A). Метод открывает путь к улучшению локальных моделей без затрат на полное переобучение.

Для Personal Wiki: если техника валидна на практике — улучшение Qwen3-8B без переобучения при ограничении RTX 3070 8GB становится рабочим сценарием.
Источник → Hugging Face, 13.04
